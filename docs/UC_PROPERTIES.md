# Unity Catalog Properties Reference

Specification for Databricks Unity Catalog table properties generated by `dqar-client-kit`.

## Overview

dqar-client-kit generates UC properties in two formats:

1. **JSON format** — for Databricks SDK programmatic loading
2. **SQL DDL format** — for manual or script-based loading

Both contain identical metadata. Choose the format that fits your workflow.

---

### Loading with Databricks SDK

```python
from databricks.sdk import WorkspaceClient
import json

client = WorkspaceClient()

with open("uc_properties.json", "r") as f:
    uc_data = json.load(f)

for table in uc_data["tables"]:
    catalog = "main"
    schema = uc_data["schema"]
    table_name = table["table_name"]
    properties = table["properties"]
    
    client.catalogs.update_properties(
        name=f"{catalog}.{schema}.{table_name}",
        properties=properties
    )
    print(f"Updated {catalog}.{schema}.{table_name}")
```

## Property Reference

### Core Lineage Properties

| Property | Type | Description |
|---|---|---|
| `dqar.data_lineage` | string | Name of ingest pipeline version (e.g., "ingest_pipeline_v2.1") |
| `dqar.source_feeds` | string | Semicolon-separated list of source feed IDs contributing to table |
| `dqar.ecds_ssor` | string | Semicolon-separated ECDS SSoR categories |

### Conformance Metrics (Levels 1–4)

| Property | Type | Description |
|---|---|---|
| `dqar.conformance_level_1_avg` | decimal | Average Level 1 (terminology) score (0.0–1.0) |
| `dqar.conformance_level_2_avg` | decimal | Average Level 2 (FHIR profile) score (0.0–1.0) |
| `dqar.conformance_level_3_avg` | decimal | Average Level 3 (clinical context) score (0.0–1.0) |
| `dqar.conformance_level_4_avg` | decimal | Average Level 4 (clinical plausibility) score (0.0–1.0) |

### Stability Property (Level 5)

| Property | Type | Description |
|---|---|---|
| `dqar.level_5_stability` | enum | PASS or FAIL (anomaly detection) |

### PIQI Quality Dimensions

| Property | Type | Description |
|---|---|---|
| `dqar.piqi_usability_avg` | decimal | Usability dimension (0.0–1.0) |
| `dqar.piqi_plausibility_avg` | decimal | Plausibility dimension (0.0–1.0) |
| `dqar.piqi_comparability_avg` | decimal | Comparability dimension (0.0–1.0) |
| `dqar.piqi_stability` | enum | PASS or FAIL |

### Metadata

| Property | Type | Description |
|---|---|---|
| `dqar.last_conformance_check` | timestamp | ISO 8601 timestamp of assessment run |
| `dqar.record_count` | integer | Number of records validated in this table |
| `dqar.plan_engagement_id` | string | Engagement ID from manifest |
| `dqar.validated_by` | string | Tool and version (e.g., "dqar-client-kit v1.0.0") |

## Loading Properties with Terraform

### terraform.tfvars

```hcl
catalog_name = "main"
schema_name  = "dqar"
plan_engagement_id = "UC1-20251014-acme"

tables = {
  "observation_fhir_flat" = {
    conformance_level_1_avg = 0.9850
    conformance_level_2_avg = 0.9800
    conformance_level_3_avg = 0.9900
    conformance_level_4_avg = 0.9750
    level_5_stability = "PASS"
    source_feeds = "lab-vendor-x-daily;ehr-epic-daily"
    data_lineage = "ingest_pipeline_v2.1"
  }
}
```

### main.tf

```hcl
resource "databricks_catalog" "main" {
  name = var.catalog_name
}

resource "databricks_schema" "dqar" {
  catalog_name = databricks_catalog.main.id
  name         = var.schema_name
}

resource "databricks_table_properties" "dqar_tables" {
  for_each = var.tables

  catalog_name = databricks_catalog.main.id
  schema_name  = databricks_schema.dqar.id
  table_name   = each.key
  
  properties = {
    "dqar.data_lineage" = each.value.data_lineage
    "dqar.source_feeds" = each.value.source_feeds
    "dqar.conformance_level_1_avg" = tostring(each.value.conformance_level_1_avg)
    "dqar.conformance_level_2_avg" = tostring(each.value.conformance_level_2_avg)
    "dqar.conformance_level_3_avg" = tostring(each.value.conformance_level_3_avg)
    "dqar.conformance_level_4_avg" = tostring(each.value.conformance_level_4_avg)
    "dqar.level_5_stability" = each.value.level_5_stability
    "dqar.plan_engagement_id" = var.plan_engagement_id
  }
}
```

## Querying Properties

### List all DQAR properties on a table

```sql
SHOW TBLPROPERTIES dqar.observation_fhir_flat;
```

Output:
```
dqar.data_lineage                     ingest_pipeline_v2.1
dqar.source_feeds                     lab-vendor-x-daily;ehr-epic-daily
dqar.conformance_level_1_avg          0.9850
dqar.conformance_level_2_avg          0.9800
...
```

### Filter by conformance level

```sql
SELECT *
FROM system.information_schema.table_properties
WHERE schema_name = 'dqar'
  AND property_name LIKE 'dqar.conformance_level_%'
  AND CAST(property_value AS DECIMAL) < 0.90;
```

### Monitor data lineage

```sql
SELECT
  table_name,
  property_value AS source_feeds
FROM system.information_schema.table_properties
WHERE schema_name = 'dqar'
  AND property_name = 'dqar.source_feeds'
ORDER BY table_name;
```

## Version Control

UC properties files should be version-controlled in your infrastructure repo:

```
infrastructure/
├── terraform/
│   ├── main.tf
│   └── terraform.tfvars
├── uc_properties/
│   ├── UC1-20251014-acme_uc_properties.json
│   └── UC1-20251014-acme_uc_properties.sql
└── README.md
```

Track property changes over time:

```bash
git log --follow uc_properties/
```

---

## Best Practices

1. **JSON for automation**, SQL for manual review
2. **Always version properties** alongside table schema changes
3. **Document data lineage** clearly in `source_feeds`
4. **Use engagement ID** to track which assessment generated properties
5. **Set last_conformance_check** to validate freshness
6. **Monitor conformance scores** for drift over time
