## What This Project Is

The client-side component of the DQAR platform. Runs on **plan infrastructure**
against PHI-containing data. No PHI ever leaves this repo's execution environment
(except in Path C where PHI loads directly to plan-owned Aidbox under a BAA).

**Key principle:** Conformance testing runs BEFORE pseudonymization.
Running after produces false failures on HMAC hashes and false passes on
date-shifted measurement periods.

## Sister Repos

| Repo | Role |
|---|---|
| `dqar-contracts` | Shared schemas, ViewDefinitions, SQL, EXT definitions. Source of truth for the client–sandbox interface. |
| `dqar-aidbox` | Aidbox-side sandbox: S3 ingest, AuditEvent metadata, SQL on FHIR assessment, findings report. |

**Dependency:** `dqar-contracts>=1.0.0,<2.0.0` is installed into this repo's venv.
During development: `pip install -e ../dqar-contracts`.
In CI/prod: install from private package registry.

**Never import from `dqar-aidbox` directly.** The interface between the two repos
is the egress package validated against `dqar_contracts.schemas.egress_package_schema_path()`.

## Six Assessment Stages

```
CLIENT ENVIRONMENT (PHI present)

  Stage 1a   Bulk FHIR API conformance preflight
  Stage 1a2  Bulk FHIR extract packaging conformance
  Stage 1b   NDJSON structural conformance testing
  Stage 1c   FHIR R4 + US Core 6.1.0 conformance testing

PHI BOUNDARY (Path B only)

  Stage 2    PHI redaction + pseudonymization (Docker stack)
             → egress package ships to dqar-aidbox

  Stage 3+   Lives in dqar-aidbox
```

## Development Principles

1. **PHI never enters Indicina infrastructure** — all client-side stages run on plan infrastructure.
2. **Conformance testing before pseudonymization** — the fixed stage order is not negotiable.
3. **`shared/engagement.py` is a shim** — do not add new logic there; import from `dqar_contracts.shared` directly.
4. **The egress package is the only output** — nothing else crosses to `dqar-aidbox`.
5. **`dqar-contracts` is pinned at major version** — breaking changes bump to v2.0.0 and require explicit update.
6. **Unknown source-type is a finding, not a suppressed error** — this logic lives in `dqar-aidbox`.
